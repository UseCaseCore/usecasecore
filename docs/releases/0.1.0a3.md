# 0.1.0a3

UseCaseCore 0.1.0a3 turns the project from a conceptual alpha into a concrete Python backend example.

## Highlights

- FastAPI + SQLAlchemy inventory demo
- POST /inventory/move endpoint
- Transaction-safe MoveInventory use case
- SQLAlchemy persistence adapters
- Idempotency replay without duplicate writes
- Audit and outbox rows written with the mutation
- Integration tests proving behavior

## Install

```bash
pip install usecasecore==0.1.0a3
```

## What Changed

This alpha adds a realistic FastAPI + SQLAlchemy example that keeps the route thin and moves the business mutation into a UseCaseCore execution boundary.

The example wires together:

- Pydantic request and response schemas
- SQLAlchemy tables for inventory balances and movements
- SQLAlchemy-backed idempotency, audit, and outbox records
- A transaction manager around the mutation
- Tests for success, idempotency replay, invalid quantity, insufficient inventory, and missing destination

## Not Included

This release does not publish new core runtime behavior. The FastAPI + SQLAlchemy work is an example layer for proving how the existing core fits into a real Python backend stack.
